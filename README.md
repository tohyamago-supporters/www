# 遠山郷応援会 ウェブサイト (tohyamago.org)

長野県南信州の秘境「下栗の里」をはじめとする遠山郷の自然・文化・暮らしを守り、後世に豊かな社会を残すために活動する **一般社団法人遠山郷応援会** の公式ウェブサイトです。

> このリポジトリの開発・運用ルールの詳細（技術スタック、ディレクトリ構造、デプロイ手順、ページ一覧、記事の追加方法など）は [`CLAUDE.md`](./CLAUDE.md) にまとめています。本 README は **サイトの目的・ビジョン・デザイン共通ルール・今後のロードマップ** を示すものです。

---

## 1. 私たちが目指すもの (Mission / Vision)

### ミッション

> **「秘境の息遣いと、守る人の想いを伝える」**

人口減少や資本流出によって衰退しつつある遠山郷において、

- 地域経済の基盤となる **農地・山林の維持保全**
- 地域内にある価値の **可視化・事業化**
- 地域の **魅力と課題の情報発信**

を行い、**自律継続的に人口の維持と経済活動の発展が行われる仕組み** をつくることを目的としています（→ [活動趣旨](./src/pages/purpose.astro)）。

地域住民だけでなく、都市部に住む人々が地域の課題を「自らに係るもの」として主体的に取り組める ―― そんな **地域内外で無理なく協力し合える環境** をウェブサイトを通じて広げていきます。

### サイトの役割

このサイトは、単なる団体紹介ページではなく、次の3つの機能を担います。

1. **記録 (Archive)** — 2018年から続く下栗芋・大豆・蕎麦・茶の作付けや地域交流の歩みを、写真と短い言葉で残し続ける（→ [`src/content/posts/`](./src/content/posts/)）。
2. **接点 (Connect)** — ボランティア・入会・成果品購入・寄付といった「関わりしろ」への入口を、迷わず提供する。
3. **支援 (Support)** — 都市部の余剰資本を地域へ還元する手段（会費・寄付・物販）を、安心して使える形で提供する。

### これまでの歩み（大切にしている流れ）

本サイトの記事は、Facebook 風の素朴な近況投稿として 2018 年から積み重ねられてきました。

- **2018〜2019**: 下栗芋・大豆の収穫、種まきなど季節の便り。短く飾らない言葉が中心。
- **2020〜2022**: 定点観測機器 (AgriRecorder) によるタイムラプス、ボランティアとの「ゆるいつながり」づくり、コロナ禍での試行錯誤。
- **2023〜2024**: 竹パウダーによる土壌改良など栽培の工夫、お茶の手摘み（一芯三葉）への挑戦と数量限定販売。
- **2025〜2026**: 新規参加者を交えた蕎麦の収穫・脱穀、茶畑の整枝、地域行事への参加と交流。

> リデザインにあたっても、この **「飾らず、続いてきた記録」** の温度感は失わないことを最優先とします。新しいデザインは、過去の歩みをより読みやすく・たどりやすくするためのものです。

## 2. ターゲットとゴール

サイト構成は **「誰に来てほしいか」「何をしてほしいか」** から逆算します。本サイトは主に **これから関わる新規来訪者** に向けた入口です（既に参加・寄付してくれている人への深い情報発信は、将来のマイページ — ロードマップ Phase 5 — に委ねます）。

### 想定ターゲット

| ターゲット                   | 像                                      | このサイトでの主目的                  |
| ---------------------------- | --------------------------------------- | ------------------------------------- |
| **ボランティア参加見込み**   | 比較的若い層（20〜40 代）。これから参加 | 農作業・イベントへの初参加 → 継続参加 |
| **支援見込み（購入・寄付）** | 富裕層・高齢層                          | 賛同しての成果品購入、寄付・入会      |

### ゴール優先順位

1. **継続的な（2 回以上）活動参加者を増やす ＝ コミュニティの拡大**（最優先）
2. 活動に賛同する気持ちでの **ショップ購入**
3. **継続寄付する会員**数の増加

> まずは **金額よりコミュニティの大きさ** を目指します。サイトの全導線は、この優先順位（参加 ＞ 購入 ＞ 寄付）の順に目立たせます。

### 「関わりしろ」の階段

来訪者を一足飛びに会員化しようとせず、軽い一歩から段階的に深い関与へ導きます。

```
知る（趣旨・始まり物語・近況）
  → 参加する（はじめての方へ・農作業カレンダー → activo 応募）   ← 最優先
    → 賛同する（成果品を買う）
      → 支える（継続寄付・入会）
```

## 3. サイト構成（情報設計とユーザー導線）

> 実装者向けの具体（ルート表・コンポーネント・データ構造・実装順序）は [`CLAUDE.md`](./CLAUDE.md) の「サイト構成（情報設計）」「新規コンテンツの仕様」「実装ステップ」に記載。

### 設計原則

- **来訪者の目的で章立てする**（団体組織図ではなく「やりたいこと」で分ける）。
- 各ページに **次の一歩（CTA）** を必ず置き、ゴール優先順位の高い「参加する／寄付」へいつでも戻れるようにする。
- 現状のコンテンツ（趣旨・入会案内・定款・公告・特商法・近況記事）は **すべて残す**。掲載場所・導線は再編する。
- **電子公告のパス `/public_notices` は法人登記に記載されているため変更しない**（最重要制約）。

### グローバルナビ（ジャーナリー導線）

| ナビ項目         | 内包                                | 主ターゲット     | 役割                     |
| ---------------- | ----------------------------------- | ---------------- | ------------------------ |
| 活動を知る       | 活動趣旨 / 始まり物語 / 近況        | 参加・支援       | 共感と理解を醸成         |
| **参加する**(主) | はじめての方へ / 農作業カレンダー   | 新規ボランティア | 最優先ゴールへの最短導線 |
| 成果品           | 成果品紹介 →（外部）ショップ        | 購入支援者       | 賛同購入                 |
| 支える           | 寄付 / 入会案内                     | 寄付・会員支援者 | 継続的支援               |
| （フッター）     | 定款 / 公告 / 特商法表記 / 法人概要 | —                | 法令順守・信頼性         |

かつての左下フローティング「法令」(`RouterMenu`、旧「目次」) は廃止し、ジャーナリー導線をヘッダー (`SiteHeader`)、法令系の文書（定款 / 公告 / 特商法表記）と法人概要をフッター (`SiteFooter`) へ収れんしました（Phase 2）。

### 新規ページの狙い

| ページ                             | 狙い（どのゴールに効くか）                                                                                                  |
| ---------------------------------- | --------------------------------------------------------------------------------------------------------------------------- |
| **始まり物語** (`/story`)          | 初期エピソードで活動の性質・温度感を伝え、共感を生む。理念（趣旨）と参加の間を埋める読み物。（参加・支援）                  |
| **農作業カレンダー** (`/calendar`) | 作物 × 作業時期をガント表示＋地域イベント重畳。「この時期に参加したい」を引き出す。運営の年間台帳も兼ねる。（参加・最優先） |
| **はじめての方へ** (`/join`)       | 初参加の不安（アクセス・服装・持ち物・流れ・FAQ）を解消し、activo 応募へ送客。継続参加の入口。（参加・最優先）              |
| **成果品紹介** (`/products`)       | 下栗芋・茶・蕎麦・大豆を作り手の物語付きで紹介し、賛同購入（shop）へ。（購入支援）                                          |
| **寄付** (`/support`)              | 単発／継続寄付の案内。Stripe 導線は Phase 4。（寄付支援）                                                                   |

### 農作業カレンダーについて

本サイト独自の目玉機能。**二つの役割**を一つの画面で満たします。

1. **参加意欲の喚起** — 「いつ・どの作物の・どんな作業に参加できるか」を一望でき、ボランティア歓迎の作業を強調。各作業から参加導線へ送る。
2. **運営の確認用台帳** — 作付け・作業・地域行事の年間スケジュールを、管理者が 1 ファイルで更新できるデータとして保持。

作物の作業（毎年循環・月粒度）と地域イベント（特定日）をデータ化し、横軸＝月・縦軸＝作物のガントチャートで描画、地域イベントを重ねて表示します。詳細なデータ構造は [`CLAUDE.md`](./CLAUDE.md) を参照。

### さらに追加を提案するコンテンツ

ゴール（特に最優先＝継続参加）に効く順で提案します。優先度は段階導入の目安です。

| 提案                                   | 効くゴール       | 優先 | 概要                                                                                 |
| -------------------------------------- | ---------------- | ---- | ------------------------------------------------------------------------------------ |
| **参加者の声 / 体験談**                | 継続参加         | 高   | 20〜40 代の不安払拭と「自分も」を促す社会的証明。`/join` 内 or 独立ページ。          |
| **アクセス案内**                       | 継続参加         | 高   | 秘境ゆえ移動が参加の最大障壁。経路・所要時間・宿泊・送迎を明示。`/join` に内包可。   |
| **定点観測（AgriRecorder）の畑ライブ** | 継続参加・愛着   | 中   | 既存 IoT のタイムラプス/近影で「育ち」を見せる engagement フック。カレンダーと連動。 |
| **活動の実績・数字（インパクト）**     | 寄付・購入       | 中   | 作付面積・参加者数・継続年数等。支援者（富裕・高齢層）の信頼を醸成。                 |
| **ニュースレター / SNS フォロー導線**  | 全ゴール（再訪） | 中   | 今すぐ動けない層を温め、再来訪・将来の参加へつなぐ。                                 |
| **プライバシーポリシー**               | 信頼・法令       | 中   | 会員・寄付・問い合わせの個人情報取扱い。Phase 4/5 で必須。                           |

> いずれも静的構成のまま実装可能。決済・会員（Stripe / Clerk）に依存するものはロードマップ Phase 4/5 に合わせる。

## 4. デザインシステム

リデザインの方向性は **「秘境の息遣いと、守る人の想いを伝える」**。広大な自然の空気感（余白）と、土に根ざした人の手仕事の温かさ（テラコッタ）を両立させます。

> 現行サイトは green / lime + 三角形 (▲) モチーフで構成されています。新デザインシステムはこれを段階的に置き換える指針であり、移行はロードマップ Phase 1〜2 で行います（[移行方針](#7-移行方針現行--新デザイン)参照）。

### 2.1 カラーパレット

Tailwind のテーマトークン（`@theme`）として定義し、`bg-base` のようなユーティリティで扱えるようにします。

| トークン                     | 値        | 用途                                                                                     |
| ---------------------------- | --------- | ---------------------------------------------------------------------------------------- |
| `base` (ミストホワイト)      | `#fdfaf3` | 雲・霧をイメージしたサイト全体のメイン背景色                                             |
| `surface`                    | `#ffffff` | カードやコンテンツエリアの背景色                                                         |
| `primary` (ディープグリーン) | `#1b3022` | 深い森。見出し・重要テキスト                                                             |
| `body` (グレー)              | `#374151` | 一般的な本文                                                                             |
| `accent` (テラコッタ)        | `#a65f3b` | 土・茶畑。ボタン・目立たせるバッジ（白文字とのコントラスト比 約 4.86:1 で WCAG AA 達成） |

### 2.2 タイポグラフィ

| 役割                   | フォント                           | 意図                       |
| ---------------------- | ---------------------------------- | -------------------------- |
| 見出し・キャッチコピー | `font-serif`（Noto Serif JP 想定） | 情緒的・伝統的・誠実       |
| 本文                   | `font-sans`（標準ゴシック）        | 読みやすさ重視             |
| 数字・英字アクセント   | `font-mono`                        | モダンで活動的なニュアンス |

ベースのフォントウェイトは light（現行 `:root { font-weight: light }` を踏襲）。

### 2.3 レイアウトとスペーシング

- **コンテナ幅**: 最大 `max-w-6xl` / `max-w-5xl`、`mx-auto` で中央揃え。左右に必ず `px-4`〜`px-6`。
- **余白（空気感）**: セクション間は `py-16`〜`py-24` を推奨。秘境の広大さを表現するため、要素を詰め込みすぎない。
- **レスポンシブ**: モバイルファースト。PC 向けは `md:` / `lg:` プレフィックス。

### 2.4 UI コンポーネント規則

- **CTA ボタン**: 丸み（`rounded-full` / `rounded-xl`）を基本に。
  ```html
  <a
    href="/donate"
    class="bg-accent text-white px-8 py-3 rounded-full font-bold transition-all hover:opacity-90 hover:shadow-lg"
    >寄付する</a
  >
  ```
- **カード（記事・商品）**: わずかなシャドウ＋ホバーで浮き上がる。
  ```html
  <article
    class="bg-surface rounded-2xl shadow-sm border border-gray-100 overflow-hidden transition-transform hover:-translate-y-1 hover:shadow-md"
  >
    …
  </article>
  ```

### 2.5 コーディング必須要件

- セマンティックタグ（`<header>` / `<main>` / `<section>` / `<article>` / `<footer>`）を正しく使う。
- 画像には必ず適切な `alt` を設定する（※現行の記事画像は `alt=""` の装飾扱いが多いため、リデザイン時に意味のある代替テキストを検討）。
- コードはシンプルに保ち、不必要に深いネストを避ける。
- インタラクティブ部品は Astro Islands（Vue 3、必要に応じて React）。静的に描けるものは Astro コンポーネントで。

### 2.6 アクセシビリティ・パフォーマンス指針

- コントラスト比は WCAG AA を満たす（特にテラコッタ上の白文字、グレー本文）。
- 画像は Astro の `<Image>` で最適化。記事画像は遅延読み込み。
- キーボード操作・フォーカスリングを確保（ナビ、決済導線、フォーム）。

## 6. ロードマップ

段階的に進めます。各フェーズは独立してリリース可能な単位とし、**「これまでの記録の温度感」を壊さない** ことを通底ルールとします。

> 凡例: ⬜ 未着手 / 🔄 進行中 / ✅ 完了

**現在地（2026-06 時点）**: Phase 0・1 完了。Phase 2 はナビ・主要ページ（`/story` `/join`、トップ刷新、固定ページ）と近況一覧のダイジェストカード化まで完了。Phase 3 はカレンダー・成果品紹介に加え、記事個別ページ `/news/[slug]` と年別・季節アーカイブ `/news/archive`（日付から判定する春夏秋冬フィルタ付き）まで完了し、記事スキーマ拡張（作物カテゴリ等）・追加コンテンツ・SEO が残課題。Phase 4 以降（決済・認証・運用）は未着手で、`/support` は案内のプレースホルダのみ。次の主眼は **静的構成で残る Phase 2／3 の仕上げ** と、**Phase 4 の前提となるハイブリッド化**。

## 7. 移行方針（現行 → 新デザイン）

リデザインは **一括置換ではなく段階移行** で進めます。

1. **トークン先行**: まず Tailwind テーマに新カラー／フォントを追加し、新旧クラスを共存させる。
2. **プリミティブ整備**: Button / Card / Container などの共通部品を新システムで作り、ページごとに差し替える。
3. **ページ単位で適用**: トップ → 固定ページ → 記事の順に新デザインへ。各段階で本番反映可能に保つ。
4. **動的化は決済直前**: SSR / Functions 化は Stripe 導入（Phase 4）の直前にまとめて行い、それまで静的構成の利点（速さ・堅牢さ）を維持する。
5. **記録の保全**: 既存記事の frontmatter（`date` / `images` / `sourceUrl`）との後方互換を必ず保つ。

## 8. 開発

```bash
npm ci          # 依存インストール
npm run dev     # 開発サーバー (http://localhost:4321)
npm run build   # 静的ビルド (dist/)
npm run preview # ビルド成果物のローカル確認
```

- ブランチ運用・デプロイ・ディレクトリ構造・記事追加手順の詳細は [`CLAUDE.md`](./CLAUDE.md) を参照。
- デプロイは Cloudflare ダッシュボードの GitHub 連携（Workers Builds）。`main` への push / PR 作成で自動ビルド・公開。GitHub Actions (`ci.yml`) はビルド検証のみ。
- 本番 / プレビューの環境変数は、単一 Worker のビルド変数に `<NAME>_PRODUCTION` / `<NAME>_PREVIEW` を登録し、**ビルド時にブランチ（`WORKERS_CI_BRANCH`）で出し分ける**（純粋関数 `src/buildEnv.ts`）。詳細は [`CLAUDE.md` の「環境変数の本番 / プレビュー切り分け」](./CLAUDE.md) を参照。

### 環境変数

| 変数名                                             | 用途                                    | 種別             | 状態               |
| -------------------------------------------------- | --------------------------------------- | ---------------- | ------------------ |
| `PDFJS_EXPRESS_VIEWER`                             | PDF.js Express ビューワーライセンスキー | 変数             | 現行               |
| `GA_MEASUREMENT_ID`                                | Google Analytics 測定 ID（gtag.js）     | 変数             | 現行               |
| `STRIPE_SECRET_KEY` / `STRIPE_WEBHOOK_SECRET` ほか | Stripe 決済                             | シークレット     | Phase 4 で追加予定 |
| `CLERK_*`（公開鍵・シークレット鍵等）              | Clerk 認証                              | 一部シークレット | Phase 5 で追加予定 |

> 本サイトは静的生成のため `PDFJS_EXPRESS_VIEWER` / `GA_MEASUREMENT_ID` は**ビルド時**に必要（ランタイム変数ではない）。デプロイ時は Cloudflare の **Settings > Build「Build variables and secrets」** に `<NAME>_PRODUCTION` / `<NAME>_PREVIEW` を登録し、ビルド時にブランチで出し分ける（プレビューに出したくない変数は `_PREVIEW` を未設定に）。CI（GitHub Actions）は値に依存しないため未設定でよい（必要時のみ `vars` に素の名前で登録）。両変数とも秘匿情報ではないため変数として扱い、将来の Stripe/Clerk の秘密鍵はビルド変数に置かずランタイムのシークレットとする。
